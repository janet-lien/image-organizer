# 小红书电商静态图片整理工具设计

日期：2026-06-05

## 1. 目标

做一个本地桌面小工具，用于把小红书电商素材整理成“每篇笔记一个文件夹”。首版只做静态模式，处理 `HEIC`、`JPG/JPEG`、`PNG` 图片。工具读取照片顺序、识别场景/款式/图片角色/质量，生成可人工确认的分组方案。确认后复制图片到输出目录，并可一键把图片上传到已有飞书多维表格记录的“图片”附件字段。

首版目标不是完全替用户决定内容，而是减少大量重复整理动作，同时保证原始照片不被破坏、上传前可确认、费用和耗时可控。

## 2. 非目标

- 不处理 Live Photo 的动态视频部分。
- 不自动决定封面图。
- 不直接移动或删除原始照片。
- 不自动创建飞书多维表格记录。
- 不自动覆盖飞书已有图片字段。
- 不追求一次导入上千张时仍然无等待感；超大批量建议分批处理。

## 4. 成篇策略

用户在导入时选择一个成篇策略。工具不在策略之间自动猜测。

### 4.1 按场景成篇 / 整套展示

同一场景内，多款式、合照和少量细节组合成一篇或多篇笔记。适合盲盒、套装、系列商品展示。

### 4.2 按款式成篇 / 单款爆款

同一款式的远景、主图、角度图、细节图、功能点、材质图组合成一篇笔记。必要时可加入整套对比图。

### 4.3 同场景精选 / 氛围种草

同一场景内挑选最好看的 3-10 张图，不强求款式完整覆盖。适合氛围感笔记。

### 4.4 自定义成篇

用户指定每篇张数范围、图片角色比例、是否允许跨场景、是否允许复用。首版可以先用简单表单表达，不做复杂规则编辑器。

## 5. 图片角色与封面

AI 识别以下图片角色：

- 远景/场景图
- 主体主图
- 正面/侧面/俯视等角度图
- 细节图
- 功能点/材质图
- 多色/多款对比图
- 整套合照

AI 不识别“封面候选”。封面由用户在分组预览中手动选择。每个正式输出文件夹必须有一个封面选择步骤；若用户未选择封面，复制输出前提示补选。

输出排序规则：

1. 用户指定的封面图固定为 `01_封面`
2. 其余图片按成篇策略排序
3. 按款式成篇优先：远景/主图、角度图、细节/功能/材质、对比/合照
4. 按场景成篇优先：场景建立、各款式展示、细节、整套合照
5. 同场景精选优先：整体观感和内容节奏

## 6. 低质量与备选图片

工具识别质量问题，但不删除图片。

### 6.1 低质量判断

本地基础检测：

- 模糊
- 过暗/过曝
- 尺寸过小
- 完全重复或近似重复

AI 辅助判断：

- 构图差
- 主体被遮挡
- 角度高度重复
- 对笔记价值较低

### 6.2 处理方式

- 明显不可用：进入全局「低质量/未使用」区。
- 轻微重复或一般质量：进入相关分组的「备选」区域。
- 用户可将备选图拖回正式分组。
- 输出时默认只复制正式分组图片；若用户勾选，也可复制备选图片到文件夹内的 `备选` 子文件夹。

## 7. 复用规则

默认每张图只进入一个正式输出文件夹。

允许复用的唯一首版场景：某个款式或分组素材不足，无法达到用户设置的目标张数下限。此时工具可推荐同款图片复用，并在预览中标记“复用 1 次”。

不做“好看的封面图到处复用”规则。复用是缺图补位，不是内容增强。

## 8. 文件处理

### 8.1 输入

首版支持：

- `.heic`
- `.jpg`
- `.jpeg`
- `.png`

首版忽略：

- `.mov`
- Live Photo 动态部分
- 视频素材
- RAW 文件

### 8.2 输出

采用复制输出。原始照片目录不移动、不删除、不改名。

输出结构：

```text
输出目录/
  01/
    01_封面.heic
    02_远景.heic
    03_主图.heic
    04_角度.heic
    05_细节.heic
  02/
    01_封面.heic
    02_主图.heic
    03_细节.heic
  未使用_低质量/
    IMG_1234.heic
```

如存在同名冲突，保留排序前缀并追加短哈希。

## 9. AI 分析与成本控制

### 9.1 分层识别

为控制费用和耗时，工具不直接把原始大图全部送入高精度模型。

流程：

1. 本地生成缩略图。
2. 本地读取 EXIF、文件名、拍摄时间、尺寸。
3. 本地做基础质量和重复检测。
4. 第一轮 AI 使用缩略图识别场景、款式、图片角色。
5. 只有低置信度、边界样本、细节图再走高精度识别。
6. 每张图片按文件哈希缓存识别结果。

HEIC 图片用于 AI 识别前会先在本地转成缩略图格式。输出和飞书上传默认保留原始静态图片格式；如果后续发现飞书预览或下游发布流程对 HEIC 不友好，再增加“输出为 JPG”的可选开关。

### 9.2 成本预估

开始分析前显示预估费用。默认单批预算上限建议为 `¥10`；超过预算需用户确认。

基于 2026-06-05 查询到的 OpenAI 价格和图片 token 计费规则，使用低成本视觉模型、缩略图、缓存、分层识别时：

- 500 张：约 `¥1.4 - ¥4`
- 800 张：约 `¥2 - ¥7`
- 全部高精度识别时可能约 `¥3 - ¥9`

价格会变化；实现时应把模型价格作为配置，界面显示“估算值”而非保证值。

参考：

- OpenAI Pricing: https://platform.openai.com/docs/pricing/
- OpenAI Images and vision: https://developers.openai.com/api/docs/guides/images-vision

## 10. 耗时目标

首版以 100 张静态图为基准估算：

| 环节 | 预计耗时 | 优化要求 |
|---|---:|---|
| 导入素材 | 30 秒 - 2 分钟 | 缩略图后台生成，先显示文件列表 |
| 自动分析 | 3 - 8 分钟 | 并发调用、缓存结果、分层识别 |
| 分组预览 | 用户操作时间 | 操作不触发重新分析 |
| 复制输出 | 10 秒 - 1 分钟 | 本地文件复制 |
| 飞书上传确认 | 1 - 3 分钟 / 批次 | 只读前 N 条匹配记录 |
| 飞书上传 | 3 - 10 分钟 / 100 张 | 队列上传、失败重试 |

目标：

- 100 张：5-10 分钟内出可检查方案。
- 300 张：15-30 分钟内出可检查方案。
- 500-800 张：允许较长等待，但必须显示阶段进度、剩余时间和费用消耗。
- 超过 800 张：提示建议分批处理。

## 11. 飞书多维表格上传

### 11.1 集成目标

上传到已有飞书多维表格。每篇笔记已有一条记录，工具只写入“图片”附件字段，不创建记录，不覆盖标题、正文、话题等字段。

### 11.2 匹配规则

用户配置：

- 多维表格 app token
- table id
- view id
- 图片字段名，默认“图片”

匹配方式：

- 读取指定视图的当前记录顺序。
- 本地 `01` 文件夹匹配第 1 条记录。
- 本地 `02` 文件夹匹配第 2 条记录。
- 只上传前 `N = min(本地文件夹数, 飞书记录数)` 个。
- 多出来的本地文件夹不上传；多出来的飞书记录不处理。
- 上传前显示确认表。

确认表字段：

- 本地文件夹编号
- 本地图片数量
- 飞书记录标题
- 飞书记录正文摘要
- 当前“图片”字段状态：空 / 已有 N 张
- 操作：上传 / 跳过 / 替换 / 追加

### 11.3 已有图片字段处理

若记录图片字段为空，默认选择上传。

若记录图片字段已有内容，逐条选择：

- 跳过
- 替换
- 追加

不做全局默认覆盖。

### 11.4 上传流程

1. 上传图片作为飞书素材，获取 `file_token`。
2. 将 `file_token` 写入对应记录的“图片”附件字段。
3. 上传队列显示每张图片状态。
4. 失败项可重试，不重新分析图片。

参考：

- 飞书附件字段说明：https://open.feishu.cn/document/server-docs/docs/bitable-v1/app-table-field/attachment?lang=zh-CN
- 飞书上传素材：https://open.feishu.cn/document/server-docs/docs/drive-v1/media/upload_all?lang=zh-CN
- 飞书更新记录：https://open.feishu.cn/document/server-docs/docs/bitable-v1/app-table-record/update?lang=zh-CN

## 12. 界面结构

### 12.1 页面

1. 导入与配置页
2. 分析进度页
3. 分组预览页
4. 封面确认页或分组内封面控件
5. 复制输出结果页
6. 飞书上传确认页
7. 上传进度与结果页
8. 设置页

### 12.2 关键控件

- 文件夹选择器：源目录、输出目录
- 成篇策略分段控件
- 目标张数输入：固定张数或范围
- 费用预算输入：默认 `¥10`
- 分组卡片：每个卡片代表一个笔记文件夹
- 图片缩略图网格：支持拖拽

- 飞书匹配确认表
- 上传操作按钮：跳过/替换/追加

## 13. 核心模块

### 13.1 File Scanner

扫描输入目录，识别支持的静态图片，读取文件名、修改时间、EXIF、尺寸，生成稳定文件 ID。

### 13.2 Thumbnail and Quality Engine

生成缩略图，做基础模糊、曝光、重复检测。

### 13.3 Vision Analyzer

调用在线视觉模型识别场景、款式、图片角色、质量补充信息。输出结构化标签、置信度和解释。

### 13.4 Grouping Engine

根据成篇策略、拍摄顺序、场景、款式、图片角色、目标张数生成分组方案。

### 13.5 Review Workspace

保存用户调整：拖拽分组、选择封面、确认复用、移入/移出低质量区。

### 13.6 Exporter

按确认方案复制文件到输出目录，并按发布顺序重命名。

### 13.7 Feishu Client

读取多维表格记录、上传图片素材、更新附件字段、处理重试和错误。

### 13.8 Cache Store

缓存缩略图、图片哈希、AI 分析结果、分组草案、上传状态。用户重新打开项目时可继续。

## 14. 数据流

```text
源图片目录
  ↓
File Scanner 读取文件和元数据
  ↓
Thumbnail and Quality Engine 生成缩略图与本地质量标签
  ↓
Vision Analyzer 生成场景/款式/角色/质量补充标签
  ↓
Grouping Engine 生成分组草案
  ↓
Review Workspace 用户确认分组、封面、复用
  ↓
Exporter 复制输出到本地文件夹
  ↓
Feishu Client 可选上传到多维表格图片字段
```

## 15. 错误处理

- 不支持的格式：跳过并显示清单。
- 图片读取失败：进入错误列表，不中断整批。
- AI 调用失败：保留本地标签，标记“待重试”。
- 超预算：暂停分析，提示确认是否继续。
- 飞书认证失败：提示重新配置凭据。
- 飞书字段不存在：停止上传，提示检查字段名。
- 飞书记录数量不足：只上传前 N 个，显示跳过数量。
- 单张上传失败：记录失败原因，允许重试。
- 用户未选封面：禁止复制输出，提示补选封面。

## 16. 验收标准

- 可以导入包含 HEIC/JPG/PNG 的目录。
- 可以选择成篇策略和目标张数范围。
- 分析前显示预计耗时和预计费用。
- AI 不生成封面候选。
- 每个输出文件夹复制前必须有用户选定封面。
- 低质量图不进入正式分组，进入低质量/备选区，且用户可手动恢复。
- 原始图片不被移动、删除、重命名。
- 输出文件夹和文件按确认后的顺序命名。
- 默认每张图只进入一个文件夹；复用只在缺图补位时发生，并清晰标记。
- 可以按飞书指定视图顺序匹配已有记录。
- 只上传前 N 个可对应文件夹。
- 已有图片字段逐条选择跳过/替换/追加。
- 上传失败可重试，不需要重新分析图片。
